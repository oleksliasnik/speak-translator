# Посібник з міграції (Migration Guide)

**З версії:** `gemini-2.5-flash-native-audio-preview-12-2025`
**На версію:** `gemini-3.1-flash-live-preview`

Офіційна міграція вимагає внесення змін до налаштувань WebSocket-сесії та логіки обробки повідомлень, оскільки версія 3.1 оптимізована для наднизької затримки та синхронного виконання.

### Чек-лист змін:

1. **Model String (Ім'я моделі):**
   - **Було:** `gemini-2.5-flash-native-audio-preview-12-2025`
   - **Стало:** `gemini-3.1-flash-live-preview`

2. **Thinking Configuration (Конфігурація "Роздумів"):**
   - Модель 3.1 перейшла від кількісного бюджету до рівнів глибини роздумів.
   - **Видалити:** `thinkingBudget` (наприклад, `thinkingBudget: 1024`).
   - **Додати:** `thinkingLevel` (доступні `minimal`, `low`, `medium`, `high`).
   - *Примітка:* Для аудіо-діалогів у реальному часі рекомендується встановлювати `minimal`, щоб уникнути затримок.

3. **Server Events (Структура відповідей сервера):**
   - Модель 3.1 тепер може об'єднувати аудіо та текст в одній відповіді (масиві).
   - Одна подія `BidiGenerateContentServerContent` може містити масив `parts` з кількома елементами одночасно.
   - **Дія:** Ваш код має ітеруватися `for (const part of serverContent.modelTurn.parts)` і обробляти всі елементи всередині однієї події (а не очікувати, що там завжди тільки один елемент).

4. **Client Content vs Realtime Input (Відправка тексту клієнтом):**
   - **Було:** Текстові повідомлення від користувача відправлялися через `send_client_content`.
   - **Стало:** `send_client_content` тепер дозволено **тільки** на самому початку для передачі початкової історії контексту (при увімкненому `initial_history_in_client_content`). 
   - **Дія:** Під час активної сесії відправляйте текстові вводи користувача через `send_realtime_input` (аналогічно до того, як відправляєте медіа-чанки аудіо/відео, але з полем `text`).

5. **Turn Coverage (Покриття ходу для Відео):**
   - **Було:** Значення за замовчуванням `TURN_INCLUDES_ONLY_ACTIVITY`.
   - **Стало:** Значення за замовчуванням `TURN_INCLUDES_AUDIO_ACTIVITY_AND_ALL_VIDEO`.
   - **Дія:** Це означає, що якщо ви відправляєте постійний відеопотік, модель буде обробляти та тарифікувати всі відеокадри, навіть якщо користувач мовчить. Щоб заощадити кошти, рекомендується відправляти відеокадри тільки в ті моменти, коли мікрофон фіксує голос (Audio Activity).

6. **Async Function Calling (Асинхронний виклик функцій):**
   - У 3.1 тимчасово **вимкнено** підтримку асинхронних викликів.
   - Виклик функцій (`Tool Calling`) тепер суворо **синхронний**. Модель повністю призупиняє роботу і не буде генерувати ні аудіо, ні текст, поки ви не надішлете `BidiGenerateContentToolResponse` назад у сокет.

7. **Proactive Audio & Affective Dialogue (Проактивне та емоційне аудіо):**
   - Ці експериментальні функції **не підтримуються** в Gemini 3.1 Flash Live Preview.
   - **Дія:** Якщо у вашому `setup` повідомленні є конфігурації проактивного або емоційного аудіо, повністю видаліть їх.

8. **Thought Signatures (Опціонально):**
   - Якщо ваш додаток зберігає історію роздумів (reasoning context) між сесіями API, модель 3.1 надсилатиме зашифрований `thoughtSignature`. Ви зобов'язані повертати його назад серверу при наступному підключенні, інакше отримаєте помилку 400. Якщо ви не зберігаєте історію між відключеннями, це можна ігнорувати.
